TECHNICAL INTERVIEW CHECKLIST

PROBLEM CLARIFICATION

Restate the problem in my own words to confirm understanding
Ask about input constraints (size, range, data types)
Confirm expected output format and requirements
Question potential edge cases:
  • Empty input
  • Maximum/minimum values
  • Duplicate elements
  • Invalid input
Clarify any ambiguities before proceeding

QUESTIONS TO ASK THE INTERVIEWER

Team/Collaboration Questions

"How do engineers and product/design teams collaborate at Airbnb?" (direct team question)
"What's the onboarding process like for new engineers?"
"How does your team make technical decisions when there are different opinions?"

Technical/Role Questions

"What kinds of technical challenges is your team currently facing?" (direct team question)
"How does your team handle technical debt and code quality?"

Company Questions

"How has Airbnb's engineering culture evolved since you joined?" (personal experience question)

SOLUTION PLANNING

Consider multiple approaches before selecting one
Explain the tradeoffs between different solutions
Select appropriate data structures with justification
Outline algorithm steps before coding
Discuss time and space complexity of planned approach
Confirm approach with interviewer before implementation

TESTING & VERIFICATION

Test with a simple example first
Verify each part of the solution incrementally
Test with edge cases:
  • Empty input
  • Single element
  • Very large input
  • Duplicates/special cases
Step through code execution manually with an example
Debug any issues methodically
Verify final solution produces correct output

INTERVIEW RED FLAGS TO AVOID

Never criticize previous employers or colleagues
Don't claim to know something you don't - admit knowledge gaps honestly
Avoid absolute statements ("always," "never," "impossible")
Don't argue defensively with the interviewer or reject hints
Never give up completely on a problem - show perseverance
Avoid focusing solely on the solution while ignoring communication
Don't rush to code without planning or clarification
Avoid negative self-talk or expressing frustration openly
Never make assumptions about requirements without verifying
Don't oversell your solution without acknowledging limitations

CLOSING STRONG

Summarize your approach concisely
Highlight key aspects of your solution
